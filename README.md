# OpenGL Engine

Небольшой учебный проект на `OpenGL 3.3`, который разбит на отдельные
классы и модули.

## Что умеет проект

- открыть окно через `GLFW`
- создать OpenGL-контекст через `GLAD`
- загрузить шейдеры из файлов
- загрузить текстуру из картинки
- собрать меш из вершин и индексов
- нарисовать текстурированный куб
- двигать камерой через `W/A/S/D`
- поднимать камеру на `Space`
- опускать камеру на `X`
- крутить обзор мышью

## Архитектура

### main.cpp

- создаёт `Application`
- запускает `app.run()`

### Application

Файлы:
- `include/Application.h`
- `src/Application.cpp`

`Application` — верхний управляющий класс проекта.

Он хранит:
- `Window`
- `Camera`
- `Shader`
- `Scene`
- таймеры кадра `deltaTime` и `lastFrame`

Он отвечает за:
- старт приложения
- главный цикл
- обновление логики
- отрисовку кадра

Главный цикл выглядит так:
1. `update()`
2. `render()`
3. `swapBuffers()`
4. `pollEvents()`

### Config

Файл:
- `include/Config.h`

Все важные настройки проекта лежат здесь.

Секции:
- `Config::Window` — размеры окна и заголовок
- `Config::Assets` — пути к шейдерам и текстуре
- `Config::Render` — цвет фона и стартовый наклон объекта

Если нужно поменять базовые параметры проекта, начинать надо с `Config.h`.

### Window

Файлы:
- `include/Window.h`
- `src/Window.cpp`

`Window` — оболочка над `GLFW`.

Он:
- вызывает `glfwInit()`
- задаёт OpenGL `3.3 Core`
- создаёт окно
- делает контекст текущим
- загружает функции OpenGL через `GLAD`
- настраивает `glViewport`

Также он даёт простые методы:
- `shouldClose()`
- `swapBuffers()`
- `pollEvents()`
- `getNativeWindow()`

### Camera

Файлы:
- `include/Camera.h`
- `src/Camera.cpp`

`Camera` хранит:
- позицию
- направление взгляда
- вектор вверх
- `yaw`
- `pitch`
- `fov`

Она умеет:
- читать клавиатуру
- читать мышь
- строить `view`-матрицу
- строить `projection`-матрицу

Текущее управление:
- `W` — вперёд
- `S` — назад
- `A` — влево
- `D` — вправо
- `Space` — вверх
- `X` — вниз
- мышь — обзор

### Scene

Файлы:
- `include/Scene.h`
- `src/Scene.cpp`

`Scene` — это уже готовая сцена.

Он объединяет:
- `Mesh`
- `Texture`
- `model`-матрицу

То есть объект знает:
- что рисовать
- чем рисовать
- как он расположен в мире

### Geometry

Файлы:
- `include/geometry/MeshData.h`
- `include/geometry/MeshLibrary.h`
- `include/geometry/CubeData.h`
- `src/geometry/MeshLibrary.cpp`
- `src/geometry/CubeData.cpp`

Этот слой нужен, чтобы большие массивы вершин и индексов не жили в `main.cpp`.

- `CubeData.cpp` хранит сырые данные куба
- `MeshData` задаёт общий формат меша
- `MeshLibrary` отдаёт готовые меши через функции вроде `Geometry::cube()`

Это значит, что форму можно переключать одной строкой, если потом добавить
другие фигуры.

### Mesh

Файлы:
- `include/Mesh.h`
- `src/Mesh.cpp`

`Mesh` отвечает за загрузку геометрии в видеокарту.

Он:
- принимает вершины и индексы
- создаёт `VAO`, `VBO`, `EBO`
- копирует данные в GPU
- настраивает атрибуты вершины
- рисует через `glDrawElements`

Формат вершины:
- позиция `x y z`
- цвет `r g b`
- UV `s t`

### Shader

Файлы:
- `include/Shader.h`
- `src/Shader.cpp`
- `shaders/vertex.glsl`
- `shaders/fragment.glsl`

`Shader`:
- читает GLSL-код из файлов
- компилирует шейдеры
- линкует программу
- даёт удобные методы для uniform-переменных

`vertex.glsl` применяет `model`, `view`, `projection`.

`fragment.glsl` сейчас просто читает цвет из текстуры.

### Texture

Файлы:
- `include/Texture.h`
- `src/Texture.cpp`

`Texture`:
- загружает картинку через `stb_image`
- создаёт OpenGL-текстуру
- настраивает wrap и filtering
- строит mipmap'ы

Текущая текстура:
- `assets/brick.jpg`

## Как проходит один кадр

Каждый кадр происходит так:

1. Считается `deltaTime`
2. Камера обновляет движение
3. Экран очищается
4. Строится `model`-матрица куба
5. Камера даёт `view`
6. Камера даёт `projection`
7. Шейдер активируется
8. В шейдер отправляются матрицы
9. `Scene` активирует текстуру и рисует меш
10. Буферы меняются местами

## Сборка

Команды:
make
make clean

Результат:
build/engine

`makefile` уже умеет собирать вложенные `.cpp`-файлы из `src/`, так что проект
можно спокойно расширять новыми модулями и подпапками.

## Куда смотреть, если хочешь что-то менять

- настройки проекта: `include/Config.h`
- поведение приложения: `src/Application.cpp`
- управление камерой: `src/Camera.cpp`
- выбор формы: `src/geometry/`
- загрузка текстуры: `src/Texture.cpp`
- работа меша с GPU: `src/Mesh.cpp`
- GLSL-логика: `shaders/*.glsl`

## Что можно развивать дальше

- добавить `Geometry::plane()` и `Geometry::pyramid()`
- сделать несколько объектов в сцене
- добавить освещение
- добавить zoom колесом мыши
- вынести сцену в отдельный класс `Scene`
- вынести рендер в отдельный `Renderer`
